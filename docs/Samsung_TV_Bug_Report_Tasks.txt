B.  Enter Verification Code (OTP) Screen
BUG-002  Focus auto-moves to Verify button after 4th OTP digit   [High]
Screen: Enter Verification Code
Observed: After the 4th digit is entered, focus automatically jumps to the Verify button. Pressing Up on the remote brings focus back to the 4th OTP field. Pressing OK on the 4th OTP field then triggers validation instead of opening the number keypad.
Expected: Pressing OK on any OTP input field should open the number keypad. Focus must never auto-shift to the Verify button. Validation should only be triggered when the user explicitly selects the Verify button and presses OK.
Fix Tasks:
Remove the auto-focus shift that moves focus to Verify after the 4th digit is entered.
Ensure pressing OK on any OTP input field opens the number keypad (not triggers validation).
Restrict OTP validation logic to fire only when the Verify button is focused and OK is pressed.
Test Up/Down/Left/Right navigation across all 4 OTP fields and the Verify button to ensure no unexpected focus jumps.
Test with correct OTP: login should proceed only via Verify button confirmation.
Test with incorrect OTP: Invalid OTP popup should appear only via Verify button confirmation.
